# Product Requirements Document (PRD): Agent Development Kit (ADK) Web Interface

## 1. Introduction
The **Agent Development Kit (ADK) Web Interface** is a comprehensive tool designed to facilitate the development, debugging, and orchestration of AI agents built with the Google ADK. It provides a visual and interactive environment to move from single-agent setups to complex multi-agent systems.

## 2. Target Audience
- **AI Engineers & Developers:** Building and testing agents using the ADK.
- **Researchers:** Exploring agentic workflows and multi-agent interactions.
- **Students/Learners:** Following the ADK study curriculum (m01-m05).

## 3. Core Features

### 3.1. Interactive Chat Interface
- **Multi-Modal Interaction:** Support for text-based chat with agents.
- **Session Management:** Create, delete, export, and import chat sessions.
- **Real-time Streaming:** Token streaming (SSE) support for responsive agent interactions.
- **Message Editing:** Ability to edit or delete messages within a session for iterative testing.

### 3.2. Agent Orchestration & Builder (Builder Mode)
- **Visual Canvas:** A node-based diagramming tool (`ngx-vflow`) to visualize agent hierarchies and workflows.
- **Agent Types:** Support for various agent classes, including `LlmAgent`, `SequentialAgent`, `ParallelAgent`, and `LoopAgent`.
- **Tool & Callback Integration:** Visually add and configure tools (Google Search, custom functions, built-in tools) and callbacks (pre/post-agent execution).
- **Sub-Agent Management:** Easily add and nest sub-agents within workflows.
- **YAML Synchronization:** Bi-directional synchronization between the visual canvas and underlying YAML configurations.

### 3.3. Advanced Debugging & Observability
- **Trace Viewer:** Detailed execution traces showing the step-by-step logic and tool calls made by agents.
- **Event Timeline:** A chronological view of all events during an agent's execution.
- **State Inspection:** Monitor and inspect the internal state of agents and sessions.
- **LLM Request/Response Details:** Deep-dive into the raw requests and responses sent to/from the Gemini models.
- **Workflow Visualization:** Graph-based representation of agent execution paths.

### 3.4. Evaluation & Testing
- **Evaluation Tab:** Tools for running and managing evaluation sets to measure agent performance.
- **Artifact Management:** View and manage artifacts generated by agents during execution.

## 4. Technical Stack
- **Frontend:**
  - Framework: Angular 21.
  - UI Library: Angular Material.
  - Visualization: `ngx-vflow` (Canvas), `ngx-json-viewer`, `ngx-markdown`.
  - State Management: RxJS and Angular Signals.
- **Backend:**
  - Language: Python.
  - Library: `google-adk` (Agent Development Kit).
  - Models: Google Gemini family (e.g., Gemini 2.0 Flash).
  - Communication: REST API with SSE support.

## 5. User Experience (UX)
- **Responsive Layout:** Side panels for configuration and debugging that can be collapsed or resized.
- **Theme Support:** Light and Dark mode toggle.
- **Developer-Centric Design:** Focus on visibility into the "black box" of LLM execution through extensive logging and visualization.

## 6. Future Considerations
- **Enhanced Multi-Modal Support:** Native handling of image and audio inputs/outputs within the builder.
- **Collaboration Features:** Shared sessions and agent configurations.
- **Deployment Integration:** One-click deployment of agents to production environments.
